# 拡張アイデア

現在の仕様（`00_spec.md`）と設計（`02_first-plan.md`）を踏まえ、将来的に考えられる拡張の方向性をまとめる。

## 1. 検索機能の強化

- **フィルター/ソート**: 言語・スター数・更新日などでの絞り込み、ソート順の切り替え（GitHub Search API がサポート済み）
- **検索履歴**: ローカルストレージに直近の検索キーワードを保存し、サジェストとして表示
- **高度な検索**: `stars:>100 language:typescript` のような GitHub 検索構文を UI で組み立てられるフォーム

## 2. 詳細画面の拡充

- **README プレビュー**: リポジトリの README をマークダウンレンダリングして表示
- **コントリビューター一覧**: アイコン付きでトップコントリビューターを表示
- **ファイルツリー表示**: リポジトリのディレクトリ構成を閲覧

## 3. ユーザー関連画面の追加

- **ユーザー詳細画面**: オーナーアイコンからユーザープロフィール・所有リポジトリ一覧へ遷移
- **GitHub OAuth 認証**: ログインにより Rate Limit 緩和（未認証は 10req/min）、プライベートリポジトリの閲覧

## 4. UX の向上

- **ダークモード**: テーマ切り替え（CSS Modules + CSS 変数で実現しやすい）
- **お気に入り/ブックマーク**: ローカルストレージでリポジトリを保存、一覧表示
- **トレンド表示**: トップページに GitHub のトレンドリポジトリを表示

## 5. 技術的な発展

- **GraphQL + Relay への移行**: `01_tech-stack.md` に候補として記載済み。1リクエストで必要なデータだけ取得でき、効率が上がる
- **PWA 対応**: Service Worker によるオフラインキャッシュ、ホーム画面追加
- **i18n**: 多言語対応（Next.js の国際化ルーティング）
